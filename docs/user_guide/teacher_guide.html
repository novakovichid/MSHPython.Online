Особенности среды


      

        
          
Локальная работа

        
        

          
Проект является frontend-приложением, поэтому вся работа происходит на стороне пользователя. Из-за того,
            что в проекте нет серверной части, все данные также хранятся на стороне пользователя.

          
Для учебного процесса это значит, что:

          
    
            
  • В виртуальном классе у каждого ученика в его браузере будет сохраняться история его проектов.
    • 
            
  • В очном отделении у каждого ученика история использования будет очищаться перед каждым занятием.
    • 
          

          
ВАЖНО! Для этого Вам нужно в конце занятия убеждаться, что ученики закрыли
              браузер, открытый до этого в режиме "Инкогнито". Совместите это с напоминанием выйти из
              Информатикса.

        

      


      

        
          
Целевые задачи

        
        

          
Среда создана для решения двух типов задач:

          
    
            
  • Консольные задачи на Python с возможностью использования ввода-вывода и import базовых библиотек и
              модулей внутри проекта;
    • 
            
  • Задачи исполнителя Черепашка с использованием библиотеки turtle с поддержкой import и обработкой
              клавиатурных событий.
    • 
          

        

      


      

        
          
Способы отправки решения

        
        

          
Предусмотрено два основных способа отправки решения: ссылкой на проект
            и архивом файлов. Подробнее - в сценариях.

        

      

    


    

      
Сценарии


      

        
          
Как отправляется решение

        
        

          

            
              
По ссылке

            
            

              
Каждая ссылка, создаваемая в МШПайтон содержит в себе хеш, в котором зашито полное содержимое проекта:
                разбивка на модули и программный код. Это причина, по которой ссылки являются такими огромными. Но это
                позволяет исопльзовать ссылки как универсальное хранилище для проектов.

              
Следите за длиной ссылки. Слишком длинную ссылку, возможно, придется отправлять как текст, а не
                ссылку. Обращайте внимание на ограничения длины ссылки в Информатиксе.

            

          

          

            
              
Архивом

            
            

              
В качестве альтернативного способа можно использовать отправку с помощью архива. В таком случае
                формируется архив, содержащий все .py файлы проекта. Формат JSON на данный момент служебный, его
                использовать не нужно.

            

          

        

      


      

        
          
Как проверить решение

        
        

          

            
              
По ссылке

            
            

              
Вам достаточно просто перейти по ссылке, отправленной Вам учеником (из Информатикса, возможно, ссылку
                придётся выделять вручную). У Вас откроется среда МШПайтон.Онлайн в режиме "Снимок".

              
Единственное его отличие от обычного - возможность сбросить проект до исходного состояния...

              

                

                  

                    
                    
                  

                

              

              
Сброс: вернуть снимок к исходному состоянию.

              
...а также сделать "Ремикс" - сохранить проект себе локально.

            

          

          

            
              
Из архива

            
            

              
На данный момент единственная возможность проверки решения из архива - использование сторонней IDE или
                ручное копирование кода.

              
Возможность импорта файлов сейчас в разработке.
